## Directory Purposes

**src/:**
- Purpose: TypeScript source code for Electron application
- Contains: Main process, preload bridge, renderer process
- Key files: `main.ts`, `preload.ts`, `renderer.ts`

**dist/:**
- Purpose: Compiled JavaScript output (generated by build)
- Contains: Compiled main.js, renderer.js, preload.js
- Generated: Yes (run `npm run build`)
- Committed: No (in .gitignore)

**otter_py/:**
- Purpose: Python package for audio transcription pipeline
- Contains: CLI entry, component registry, transcriber/post-processor implementations
- Key files: `transcribe.py`, `pipeline_registry.py`

**otter_py/pipelines/transcribers/:**
- Purpose: Container for transcription/ASR implementations
- Contains: Modules implementing different transcription approaches (Whisper-based)
- Pattern: Each module registers itself via @register_transcriber decorator on import

**otter_py/pipelines/postprocessors/:**
- Purpose: Container for transcript post-processing implementations
- Contains: Word timing adjustment, cleaning, normalization modules
- Pattern: Each module registers itself via @register_postprocessor decorator on import

**otter_py/sample_specs/:**
- Purpose: Pre-built pipeline configuration files (JSON)
- Contains: Different combinations of transcriber + post-processor selections
- Examples:
  - `default_spec.json`: Default pipeline (currently faster_whisper with post-processors)
  - `fw.json`: faster_whisper only
  - `wx_asw_cwt.json`: WhisperX with post-processors

**assets/:**
- Purpose: Application static assets (images)
- Contains: `icon.png` (window icon), `BannerIcon.png` (header logo)

**doc/:**
- Purpose: User-facing and conceptual documentation
- Contains: Design documents, user stories, editing concepts

**test_data/:**
- Purpose: Sample audio files for testing and demonstration
- Contains: Short .wav files for development/testing

**.venv/:**
- Purpose: Python virtual environment (development only)
- Contains: Python executable and installed packages
- Committed: No (in .gitignore)

**.planning/:**
- Purpose: GSD analysis output and planning documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md

## Key File Locations

**Entry Points:**
- `src/main.ts`: Electron main process entry, creates window, sets up IPC
- `src/renderer.ts`: Browser renderer, compiles to `dist/renderer.js`, runs in HTML context
- `index.html`: Loads compiled renderer.js and WaveSurfer library
- `otter_py/transcribe.py`: Python CLI, invoked by main process as subprocess

**Configuration:**
- `tsconfig.base.json`: Base TypeScript compiler options
- `tsconfig.main.json`: Extends base, targets src/main.ts → dist/main.js
- `tsconfig.renderer.json`: Extends base, targets src/renderer.ts → dist/renderer.js
- `eslint.config.cjs`: Linting rules for .ts files
- `package.json`: NPM scripts (build, start, lint) and dependencies

**Core Logic:**
- `src/main.ts`: IPC handlers for file dialogs, ffmpeg/ffprobe execution, transcription spawning
- `src/preload.ts`: Bridge exposing `window.otter` API (audio, transcribe, probe, snippet, spec operations)
- `src/renderer.ts`: UI rendering (WaveSurfer setup, transcript rendering, playback synchronization)
- `otter_py/pipeline_registry.py`: Component registration and pipeline execution engine
- `otter_py/transcribe.py`: CLI argument parsing and pipeline invocation

**Styling:**
- `style.css`: Two-pane layout (waveforms left, transcript right), dark log area, transport controls

**Testing:**
- No test framework present (PoC scope)
- Test data: `test_data/` contains sample audio files

## Naming Conventions

**Files:**
- Main process: `main.ts`
- Preload script: `preload.ts`
- Renderer: `renderer.ts`
- Python modules: lowercase with underscores (`transcribe.py`, `pipeline_registry.py`)
- Post-processors: verb_noun format (`adjust_short_words.py`, `clean_word_timings.py`)

**Directories:**
- TypeScript: `src/` (source), `dist/` (compiled)
- Python packages: lowercase with underscores (`otter_py/`, `pipelines/`, `postprocessors/`)

**TypeScript Identifiers:**
- Types: PascalCase (`TranscriptWord`, `OtterApi`, `Word`)
- Functions: camelCase (`setPlayheadIndex()`, `renderTranscript()`)
- Variables/state: camelCase (`audioPath`, `words`, `selectionStart`)
- Constants: UPPER_CASE (`SEEK_EPS`, `DETAIL_PAD_BEFORE`, `WORD_REGION_COLOR`)
- IPC handlers: kebab-case (`"choose-audio-file"`, `"transcribe-audio"`, `"make-snippet"`)

**Python Identifiers:**
- Functions: snake_case (`read_spec()`, `run_pipeline()`)
- Classes: PascalCase (if used)
- Constants: UPPER_CASE (`_TRANSCIBERS`, `_POSTS`)
- Type variables: PascalCase (`Word`, `TranscriberFn`)

## Where to Add New Code

**New Feature (Audio Processing):**
- Primary code: `src/main.ts` (add new IPC handler)
- Preload bridge: `src/preload.ts` (expose handler via window.otter)
- Renderer: `src/renderer.ts` (call via window.otter, update UI)
- Example: Adding a new audio effect would follow this pattern

**New Transcriber:**
- Implementation: `otter_py/pipelines/transcribers/my_transcriber.py`
- Registration: Decorate with `@register_transcriber(...)`
- Auto-load: Add import to `otter_py/pipelines/transcribers/__init__.py`
- Config: Create spec file in `otter_py/sample_specs/my_spec.json`

**New Post-Processor:**
- Implementation: `otter_py/pipelines/postprocessors/my_processor.py`
- Registration: Decorate with `@register_postprocessor(...)`
- Auto-load: Add import to `otter_py/pipelines/postprocessors/__init__.py`
- Use: Reference by ID in pipeline spec JSON

**Utilities:**
- Shared helpers: Add to `otter_py/` root (e.g., `otter_py/audio_utils.py`)
- Browser utilities: Add utility functions in renderer.ts (grouped by concern, e.g., playback, selection)

**UI Components:**
- Static HTML: Modify `index.html` (add elements, ids for JS targeting)
- Styling: Add classes to `style.css` and reference from renderer.ts
- Interaction logic: Add event listeners and state management in `src/renderer.ts`

## Build and Compilation

**TypeScript Compilation:**
```bash
npm run build
# Runs: tsc -p tsconfig.main.json && tsc -p tsconfig.renderer.json
```
- Compiles src/main.ts → dist/main.js (targeting ES2020, strict mode)
- Compiles src/renderer.ts → dist/renderer.js (targeting ES2020, strict mode)
- Also compiles src/preload.ts → dist/preload.js

**Running the Application:**
```bash
npm start
# Builds and starts Electron with dist/main.js as entry point
```

**Python:**
- No compilation needed
- Entry point: `python -m otter_py.transcribe [list|run]`
- Virtual environment: `.venv/bin/python` (if available)
